# Fore ERP 資料庫建制說明

本專案使用 **Supabase** 作為資料庫。以下說明如何建立／更新資料庫。

---

## 一、若還沒有 Supabase 專案

1. 前往 [supabase.com](https://supabase.com) 登入或註冊。
2. 點 **New project**，選組織、輸入專案名稱與資料庫密碼，選區域（例如 Singapore）。
3. 建立完成後，到 **Project Settings → API** 取得：
   - **Project URL** → 填到 `.env.local` 的 `NEXT_PUBLIC_SUPABASE_URL`
   - **anon public** key → `NEXT_PUBLIC_SUPABASE_ANON_KEY`
   - **service_role** key（僅後端用）→ `SUPABASE_SERVICE_ROLE_KEY`（選填，portal 登入等會用到）
4. 專案內需先有 ERP 使用到的**既有資料表**（例如 `customers`、`orders`、`order_items`、`work_orders`、`product_variants`、`user_profiles` 等）。若你是從頭建立，這些表需要先依你的業務設計建立；若已有舊專案或別人提供給你的 SQL，請先執行那些 SQL 建立基礎表結構。

---

## 二、在 Supabase 執行 Migration（建議方式）

Migration 檔案放在專案 **`supabase/migrations/`**，依**檔名時間順序**執行（先執行數字較小的）。

### 方法 A：用 Supabase 網頁 SQL Editor（最簡單）

1. 登入 [Supabase Dashboard](https://supabase.com/dashboard)，選你的專案。
2. 左側點 **SQL Editor**。
3. 新增查詢，把下面兩段 SQL **依序**貼上並執行（一次一段即可）。

---

#### Migration 1：通路與訂單來源欄位（20250308）

若你的專案**已經有** `customers` 和 `orders` 表，執行這段會加上「通路下單」與「訂單來源」需要的欄位：

```sql
-- 通路商入口：客戶可設定「通路代碼」與「通路密碼」，用於 /portal 登入
ALTER TABLE customers
  ADD COLUMN IF NOT EXISTS portal_code text UNIQUE,
  ADD COLUMN IF NOT EXISTS portal_password text;

COMMENT ON COLUMN customers.portal_code IS '通路下單入口登入代碼，唯一';
COMMENT ON COLUMN customers.portal_password IS '通路下單入口密碼（建議正式環境改為雜湊儲存）';

-- 訂單來源：區分 ERP 內部建立 vs 通路 portal 下單（總覽可顯示「通路新單」）
ALTER TABLE orders
  ADD COLUMN IF NOT EXISTS source text DEFAULT 'erp';

COMMENT ON COLUMN orders.source IS 'erp: 內部建立; portal: 通路商下單';
```

---

#### Migration 2：使用回饋表（20250309）

建立「使用回饋」功能用的表：

```sql
-- 使用回饋（暫時性功能）：夥伴回報使用 ERP 時遇到的問題，方便逐筆處理
CREATE TABLE IF NOT EXISTS user_feedback (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  title text NOT NULL,
  description text,
  category text NOT NULL,
  status text NOT NULL DEFAULT '待處理',
  priority text,
  reporter text,
  internal_notes text,
  created_at timestamptz DEFAULT now(),
  updated_at timestamptz DEFAULT now()
);

COMMENT ON TABLE user_feedback IS '使用回饋／問題回報（暫時性功能）';
COMMENT ON COLUMN user_feedback.title IS '主旨';
COMMENT ON COLUMN user_feedback.description IS '問題描述';
COMMENT ON COLUMN user_feedback.category IS '類別（對應左側功能）';
COMMENT ON COLUMN user_feedback.status IS '狀態：待處理、處理中、已解決、暫緩';
COMMENT ON COLUMN user_feedback.priority IS '優先級：低、中、高';
COMMENT ON COLUMN user_feedback.reporter IS '回報人（email 或名稱）';
COMMENT ON COLUMN user_feedback.internal_notes IS '內部備註（處理方式等）';
```

### 方法 B：用 Supabase CLI（本機連線到同一個專案）

若你已安裝 [Supabase CLI](https://supabase.com/docs/guides/cli) 且已登入、連結到該專案：

```bash
# 在專案根目錄
cd c:\Users\v6667\OneDrive\Desktop\fore-erp
supabase link --project-ref 你的專案ID
supabase db push
```

`supabase db push` 會依 `supabase/migrations/` 裡的檔案順序，把尚未執行過的 migration 套用到遠端資料庫。

---

## 三、確認是否成功

- **Migration 1**：在 **Table Editor** 看 `customers` 是否有 `portal_code`、`portal_password`，`orders` 是否有 `source`。
- **Migration 2**：在 **Table Editor** 是否出現 `user_feedback` 表。

若執行時出現「某表不存在」錯誤，代表該表尚未建立，需先建立 ERP 的基礎表（例如 `customers`、`orders` 等），再執行上述 migration。
